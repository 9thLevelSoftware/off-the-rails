## Task 1: WorkshopAdapter

**Action**: Create `src/crafting/adapters/workshop_adapter.gd` to connect Fabricator to the crafting system.

```gdscript
class_name WorkshopAdapter
extends Node

## Bridges WorkshopCar/Fabricator to the crafting domain.
## Handles power state, speed modifiers, and tick delegation.

var workshop: WorkshopCar
var fabricator: Fabricator
var scheduler: JobScheduler
var pause_handler: ExpeditionPauseHandler

@export var tick_enabled: bool = true

func _ready() -> void:
    # Find workshop in parent hierarchy or via injection
    _setup_crafting_system()
    _connect_fabricator_signals()

func _process(delta: float) -> void:
    if not tick_enabled:
        return
    if not fabricator or not fabricator.is_ready_for_crafting():
        return
    scheduler.tick(delta * _get_speed_modifier())

func _get_speed_modifier() -> float:
    # Base speed from fabricator
    var base = fabricator.get_crafting_speed() if fabricator else 1.0
    # Could add profession bonuses here in future
    return base
```

Adapter should:
- Initialize JobScheduler with dependencies
- Connect to Fabricator's `fabricator_ready_changed` signal
- Pause scheduler when fabricator goes offline (power loss)
- Resume when fabricator comes back online
- Expose methods for UI to call (enqueue, cancel, get queue)
- **CRITIQUE FIX**: Pass profession_id when enqueuing recipes

Add method to get current profession:
```gdscript
func _get_player_profession_id() -> String:
    var game_state = Engine.get_singleton("GameState")
    if game_state and game_state.player_profession:
        return game_state.player_profession.id
    return ""

func enqueue_recipe(recipe: RecipeData) -> Result:
    var profession_id := _get_player_profession_id()
    return scheduler.enqueue_recipe(recipe, profession_id)
```

**Verify**: Adapter ticks scheduler when Fabricator is ready, pauses when offline, passes profession correctly.

**Done**: WorkshopAdapter created connecting Fabricator to JobScheduler with profession support.

```gdscript
class_name QueueDisplay
extends PanelContainer

## CRITIQUE FIX: UI connects ONLY to EventBus, not directly to CraftQueue.
## CraftQueue is RefCounted and its signals don't propagate reliably to UI.
## JobScheduler forwards all queue events to EventBus.

func _ready() -> void:
    _connect_to_event_bus()

func _connect_to_event_bus() -> void:
    var bus = CraftingEventBus.get_instance()
    # Job lifecycle signals
    bus.job_queued.connect(_on_job_queued)
    bus.job_started.connect(_on_job_started)
    bus.job_progress.connect(_on_job_progress)
    bus.job_completed.connect(_on_job_completed)
    bus.job_cancelled.connect(_on_job_cancelled)
    # Queue structure signals (CRITIQUE FIX: forwarded from JobScheduler)
    bus.queue_job_added.connect(_on_queue_job_added)
    bus.queue_job_removed.connect(_on_queue_job_removed)
    bus.queue_paused.connect(_on_queue_paused)
    bus.queue_resumed.connect(_on_queue_resumed)

func _update_display() -> void
func _format_time(seconds: float) -> String  # "1:30" format
```

**CRITIQUE FIX**: Display updates reactively via EventBus signals ONLY. Never hold direct reference to CraftQueue.

**Verify**: Queue display updates when jobs added/completed, progress bar animates.
