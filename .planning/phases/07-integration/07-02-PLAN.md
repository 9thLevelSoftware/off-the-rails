## Task 1: Unify ContentRegistry (CRITICAL)

**Problem:** ModLoader creates `ContentRegistry.new()` at line 51, GameState creates separate `ContentRegistry.new()` at line 71. Mod content loads into ModLoader's registry but crafting uses GameState's registry = mods invisible.

**Solution:** GameState should use ModLoader's ContentRegistry instead of creating its own.

**Action:**
1. Open `src/autoloads/game_state.gd`
2. Modify `get_content_registry()` to return ModLoader's registry:
   ```gdscript
   func get_content_registry() -> ContentRegistry:
       # Use ModLoader's registry (single source of truth for all content including mods)
       if ModLoader and ModLoader.is_ready():
           return ModLoader.get_content_registry()
       # Fallback for tests or if ModLoader not ready
       if _content_registry == null:
           if not init_content_registry():
               push_warning("GameState: ContentRegistry failed to load base content")
       return _content_registry
   ```
3. This ensures GameState.get_item_data(), get_recipe_data() etc. all see mod content

**Files:** `src/autoloads/game_state.gd`

> verification: grep -q "ModLoader.get_content_registry" src/autoloads/game_state.gd

**Done when:** GameState returns ModLoader's registry when available.

## Task 2: Wire CraftingEventBus to CraftingUI

**Problem:** `crafting_ui_requested` signal exists but nothing subscribes to show the UI.

**Solution:** HUD subscribes and manages CraftingUI visibility. Add CraftingUI as sibling in main scene's UILayer (not as child of HUD CanvasLayer to avoid layer conflicts).

**Action:**
1. Open `src/main.tscn`
2. Add CraftingUI scene instance under UILayer (sibling to HUD):
   ```
   [node name="CraftingUI" parent="UILayer" instance=ExtResource("crafting_ui_scene")]
   ```
3. Open `src/main.gd`
4. Add reference and connection:
   ```gdscript
   @onready var _crafting_ui: CraftingUI = $UILayer/CraftingUI
   
   func _ready() -> void:
       # ... existing code ...
       # Connect crafting UI signal
       CraftingEventBus.get_instance().crafting_ui_requested.connect(_on_crafting_ui_requested)
   
   func _on_crafting_ui_requested(requester) -> void:
       if requester is WorkshopAdapter:
           _crafting_ui.open(requester)
   ```

**Files:** `src/main.tscn`, `src/main.gd`

> verification: grep -q "crafting_ui_requested.connect" src/main.gd

**Done when:** CraftingEventBus signal triggers CraftingUI.open().

## Task 3: Add WorkshopAdapter to Scene & Wire to Workbench

**Problem:** WorkshopAdapter class exists but is not instantiated anywhere. Also, workbench EquipmentInteractable is private to WorkshopSpatialAdapter.

**Solution:** 
- Add WorkshopAdapter node to workshop.tscn
- Add method to WorkshopSpatialAdapter to expose interactable by type
- Wire them in workshop_layout_controller after equipment is rendered

**Action:**

**Step 3a: Add WorkshopAdapter node to workshop.tscn**
1. Open `src/train/cars/workshop/scenes/workshop.tscn`
2. Add WorkshopAdapter node as child of Workshop root:
   ```
   [node name="WorkshopAdapter" type="Node" parent="."]
   script = ExtResource("workshop_adapter_script")
   ```
3. Add ext_resource for workshop_adapter.gd

**Step 3b: Expose interactable in WorkshopSpatialAdapter**
1. Open `src/train/cars/workshop/adapters/workshop_spatial_adapter.gd`
2. Add method to get interactable by equipment type:
   ```gdscript
   ## Get an EquipmentInteractable by equipment type (e.g., "WORKBENCH")
   func get_interactable_by_type(equipment_type: String) -> EquipmentInteractable:
       for interactable in _equipment_interactables.values():
           if interactable.equipment_type.to_upper() == equipment_type.to_upper():
               return interactable
       return null
   ```

**Step 3c: Wire in workshop_layout_controller**
1. Open `src/train/cars/workshop/adapters/workshop_layout_controller.gd`
2. After equipment setup completes, wire the adapter:
   ```gdscript
   @onready var _workshop_adapter: WorkshopAdapter = $WorkshopAdapter
   
   func _ready() -> void:
       # ... existing setup code ...
       # Wire WorkshopAdapter after equipment is rendered
       call_deferred("_wire_workshop_adapter")
   
   func _wire_workshop_adapter() -> void:
       var workbench := _spatial_adapter.get_interactable_by_type("WORKBENCH")
       if workbench:
           _workshop_adapter.connect_to_workbench(workbench)
           print("[WorkshopLayoutController] Wired WorkshopAdapter to workbench")
       else:
           push_warning("[WorkshopLayoutController] No workbench found to wire")
   ```

**Files:** 
- `src/train/cars/workshop/scenes/workshop.tscn`
- `src/train/cars/workshop/adapters/workshop_spatial_adapter.gd`
- `src/train/cars/workshop/adapters/workshop_layout_controller.gd`

## Task 4: Handle Fabricator Dependency (Prototype Mode)

**Problem:** WorkshopAdapter._process() only ticks scheduler when `_fabricator.is_ready_for_crafting()`. Without Fabricator, crafting never progresses.

**Solution:** For V2 prototype, allow WorkshopAdapter to tick without Fabricator (always ready mode). Add TODO for V2.1 Fabricator integration.

**Action:**
1. Open `src/crafting/adapters/workshop_adapter.gd`
2. Modify `_process()` to work without Fabricator:
   ```gdscript
   func _process(delta: float) -> void:
       if _scheduler == null:
           return
       
       # V2 PROTOTYPE: Tick scheduler without Fabricator requirement
       # TODO (V2.1): Re-enable Fabricator check when subsystems are implemented
       var speed := 1.0
       if _fabricator and _fabricator.is_ready_for_crafting():
           speed = _fabricator.get_crafting_speed()
       _scheduler.tick(delta * speed)
   ```
3. Add comment explaining this is temporary for prototype

**Files:** `src/crafting/adapters/workshop_adapter.gd`

> verification: grep -q "V2 PROTOTYPE" src/crafting/adapters/workshop_adapter.gd

**Done when:** Crafting queue ticks without requiring Fabricator setup.
