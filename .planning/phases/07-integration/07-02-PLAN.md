<objective>
Create a persistent HUD overlay that displays game state to the player. The HUD shows escalation level during expeditions, inventory counts, and a placeholder health bar. It must update reactively when underlying state changes.
</objective>

<context>
## Prior Work (07-01)
- All systems now properly wired with signals
- GameState.inventory_changed fires on all inventory updates
- EscalationManager.escalation_changed fires on escalation updates

## Key Systems to Display
- **Escalation**: 0-100% meter with threshold markers (from EscalationManager)
- **Inventory**: Key resource counts (scrap_metal, wire, etc.)
- **Health**: Placeholder for future combat system (static bar for now)

## UI Architecture
- CanvasLayer at layer 10 (above game scenes, below menus)
- Persistent across scene transitions
- HUD node added in main.tscn or as autoload
- Uses Control nodes for responsive layout

## Design References
- `docs/design/systems/expeditions.md` — Escalation threshold descriptions
- `docs/design/data/resources.yaml` — Resource types for inventory display
</context>

<tasks>
## Task 1: Create HUD CanvasLayer and Scene Structure

**Action**: Create the base HUD scene with proper layering.

1. Create `src/ui/hud/hud.tscn` as a CanvasLayer scene
2. Set layer property to 10 (above gameplay, below menus at 20)
3. Add root structure:
   ```
   HUD (CanvasLayer)
   └── MarginContainer (full screen margins)
       ├── TopBar (HBoxContainer, anchored top)
       │   ├── HealthContainer
       │   └── EscalationContainer
       └── BottomBar (HBoxContainer, anchored bottom)
           └── InventoryDisplay
   ```
4. Create `src/ui/hud/hud.gd` script to manage connections
5. Use Control anchors for responsive positioning (top-left, top-right, bottom-center)

**Verify**:
- Scene loads without errors
- HUD visible above game content
- Layout responds to window resize

**Done when**: HUD shell exists with placeholder containers.

---

## Task 2: Implement Escalation Meter Display

**Action**: Create visual escalation meter that updates in real-time.

1. Create `src/ui/hud/escalation_meter.gd` and `escalation_meter.tscn`
2. Visual design:
   - Horizontal progress bar (0-100%)
   - Color gradient: green (0-25%) → yellow (25-50%) → orange (50-75%) → red (75-100%)
   - Threshold markers as tick marks on the bar
   - Current percentage label
3. Connect to EscalationManager:
   ```gdscript
   func _ready() -> void:
       var escalation_mgr = _find_escalation_manager()
       if escalation_mgr:
           escalation_mgr.escalation_changed.connect(_on_escalation_changed)
           escalation_mgr.threshold_reached.connect(_on_threshold_reached)
   ```
4. Hide meter when in Train scene (only show during expedition)
5. Add visual feedback on threshold crossing (brief flash or pulse)

**Verify**:
```gdscript
# In expedition:
# 1. Meter visible and starts at current escalation %
# 2. Wait for escalation to increase — meter updates
# 3. Cross threshold — visual feedback triggers
# 4. Return to train — meter hides
```

**Done when**: Escalation meter accurately reflects EscalationManager state with threshold feedback.

---

## Task 3: Implement Inventory Resource Display

**Action**: Show counts of key resources in the HUD.

1. Create `src/ui/hud/inventory_display.gd` and `inventory_display.tscn`
2. Display format: icon + count for each tracked resource
3. Track these V1 resources (from resources.yaml):
   - scrap_metal
   - wire
   - electronic_components
   - repair_kit (crafted output)
4. Layout: horizontal row of resource indicators at bottom of screen
5. Connect to GameState.inventory_changed:
   ```gdscript
   func _ready() -> void:
       GameState.inventory_changed.connect(_on_inventory_changed)
       _refresh_all_counts()
   
   func _on_inventory_changed(item_id: String, _old: int, new_qty: int) -> void:
       if item_id in tracked_resources:
           _update_resource_display(item_id, new_qty)
   ```
6. Use placeholder icons (colored rectangles) — art deferred

**Verify**:
```gdscript
# 1. Start with empty inventory — all counts show 0
# 2. Add item: GameState.add_to_inventory("scrap_metal", 5)
# 3. Display updates to show 5
# 4. Craft item that consumes scrap_metal — count decreases
```

**Done when**: Inventory display shows accurate counts that update reactively.

## Task 4: Add Health Bar Placeholder

**Action**: Create a health bar UI element for future combat system.

1. Create `src/ui/hud/health_bar.gd` and `health_bar.tscn`
2. Visual: red progress bar at top-left
3. For V1: static display at 100% (no combat system yet)
4. Expose API for future:
   ```gdscript
   var max_health: float = 100.0
   var current_health: float = 100.0
   
   func set_health(current: float, maximum: float) -> void:
       current_health = current
       max_health = maximum
       _update_display()
   
   func take_damage(amount: float) -> void:
       set_health(current_health - amount, max_health)
   ```
5. Add brief visual feedback for damage (red flash) — can test manually

**Verify**:
- Health bar renders at top-left
- `health_bar.set_health(50, 100)` updates to 50%
- `health_bar.take_damage(25)` reduces further

**Done when**: Health bar displays correctly and has working API for future combat integration.

---

## Task 5: Integrate HUD into Main Scene

**Action**: Add HUD to the game and verify it persists across transitions.

1. Modify `src/main.gd` or `src/main.tscn` to instantiate HUD
2. HUD should be added as child of Main (not Train or Expedition)
3. Verify HUD persists when transitioning between scenes
4. Add HUD reference to GameState if needed for other systems
5. Test:
   - Start in Train: HUD visible, escalation meter hidden
   - Transition to Expedition: HUD visible, escalation meter shows
   - Return to Train: HUD visible, escalation meter hides

**Verify**:
- HUD always visible regardless of current scene
- No duplicate HUD instances after multiple transitions
- HUD updates correctly in both scenes

**Done when**: HUD is integrated and persistent across the entire game session.
