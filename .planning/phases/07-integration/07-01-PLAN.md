**Action**: VERIFY (not implement) that crafted items are already wired to GameState inventory.

2. Read `src/crafting/infrastructure/inventory_repository.gd` to confirm it calls GameState.add_all_inventory()
3. If wiring exists: verify by testing (step 4)
4. If wiring is missing: add connection in JobScheduler._complete_job() or InventoryRepository

**Verify**:
```gdscript
# In debug console or test:
```

## Task 3: Wire Loot Collection to Inventory

**Action**: Connect LootContainer.container_opened signal to GameState inventory.

**IMPORTANT**: LootContainer emits `container_opened(loot_contents: Array[LootItem])`, NOT a generic `collected` signal. LootItem has properties: `item_name`, `quantity`, `item_type`.

1. Read `src/expedition/loot/loot_container.gd` — signal is at line 24
2. Read `src/expedition/loot/loot_item.gd` — data structure with item_name, quantity, item_type
3. Add inventory integration in LootContainer:
   ```gdscript
   # In LootContainer.interact() after container_opened.emit(contents):
   for item in contents:
       GameState.add_to_inventory(item.item_name, item.quantity)
   ```
   Or connect via signal handler in expedition.gd if you prefer separation
4. Container already sets is_opened = true, preventing double-pickup

**Verify**:
```gdscript
# In expedition scene:
# 1. Create test container with known contents:
#    LootItem.create("scrap_metal", 3, "material")
# 2. Interact with LootContainer
# 3. Check: GameState.get_inventory_quantity("scrap_metal") >= 3
# 4. Try to interact again — should say "Already opened"
```

**Done when**: Opening a loot container adds all its LootItem contents to inventory.
