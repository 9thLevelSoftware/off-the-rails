## Task 3: Create PlayerController Adapter
**Action**: Create `src/isometric/adapters/player_controller.gd` with:
- `class_name PlayerController extends CharacterBody2D`
- Exported properties:
  - `movement_config: MovementConfig` — Drag resource in editor
- Private vars:
  - `_animation_controller: PlayerAnimationController`
  - `_current_direction: int`
- `_ready()`: Get child references, set default movement_config if null
- `_physics_process(delta)`:
  1. Get raw input: `Input.get_vector("move_left", "move_right", "move_forward", "move_backward")`
  2. Convert to isometric: `InputConverter.wasd_to_isometric(raw_input)`
  3. Apply acceleration/friction using LERP-based smoothing:
     ```gdscript
     # Acceleration/friction algorithm (critique finding #2):
     var target_velocity := iso_direction * movement_config.walk_speed
     if iso_direction.length() > 0.1:
         # Accelerate toward target
         velocity = velocity.move_toward(target_velocity, movement_config.acceleration * delta)
     else:
         # Apply friction to stop
         velocity = velocity.move_toward(Vector2.ZERO, movement_config.friction * delta)
     ```
  4. Call `move_and_slide()`
  5. Update animation controller with `velocity` and `is_moving` bool
- Y-sort note: Godot's Y-sort uses `global_position.y` automatically after `move_and_slide()` — no manual sync needed

**Verify**:
```bash

# Acceleration/friction uses move_toward() pattern
```

**Done when**: PlayerController handles input conversion, physics movement, and animation updates with smooth acceleration.
