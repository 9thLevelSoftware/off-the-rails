`src/mods/example_item_mod/scripts/on_init.gd`:
```gdscript
extends RefCounted

## Example mod initialization script
## Demonstrates how to use ModAPI to register content programmatically

func _mod_init(api: ModAPI) -> void:
    print("[ExampleItemMod] Initializing...")
    
    # Example: Register an item via script (alternative to YAML)
    var scripted_item := {
        "id": "debug_tool",
        "name": "Debug Tool",
        "description": "A developer tool for testing. Added via script.",
        "category": "tool",
        "stack_size": 1
    }
    
    if api.register_item(scripted_item):
        print("[ExampleItemMod] Registered debug_tool via script")
    
    # Connect to game events
    EventHooks.game_ready.connect(_on_game_ready)
    EventHooks.craft_completed.connect(_on_craft_completed)
    
    print("[ExampleItemMod] Initialization complete!")

func _on_game_ready() -> void:
    print("[ExampleItemMod] Game is ready!")

func _on_craft_completed(recipe_id: String, station_id: String, item_id: String) -> void:
    if "example_item_mod" in recipe_id:
        print("[ExampleItemMod] Player crafted our item: %s" % item_id)
```

```markdown
# Mod API Reference

Complete reference for the Off The Rails modding API.

## Overview

The mod system provides three main interfaces:

1. **mod.json** - Mod manifest for metadata and file declarations
2. **ModAPI** - Typed methods for registering content
3. **EventHooks** - Signals for responding to game events

---

## mod.json Schema

Every mod requires a `mod.json` file in its root directory.

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique mod identifier (lowercase, alphanumeric, underscores) |
| `version` | string | Semantic version (X.Y.Z format) |
| `name` | string | Display name for the mod |

### Optional Fields

| Field | Type | Description |
|-------|------|-------------|
| `description` | string | Mod description |
| `author` | string | Mod author name |
| `dependencies` | string[] | Required mod IDs (loaded first) |
| `content_files` | string[] | JSON data files to load |
| `scripts` | string[] | GDScript files to execute |

### Example

```json
{
  "id": "my_awesome_mod",
  "version": "1.0.0",
  "name": "My Awesome Mod",
  "description": "Adds awesome stuff.",
  "author": "Your Name",
  "dependencies": [],
  "content_files": ["data/items.json"],
  "scripts": ["scripts/on_init.gd"]
}
```

---

## ModAPI Methods

ModAPI is passed to your mod scripts via `_mod_init(api: ModAPI)`.

### Content Registration

#### register_item(item_data: Dictionary) -> bool

Register a custom item.

**Required fields:**
- `id` (string) - Item identifier (will be prefixed with mod ID)
- `name` (string) - Display name

**Optional fields:**
- `description` (string) - Item description
- `category` (string) - Item category: "resource", "consumable", "tool", "quest_item"
- `stack_size` (int) - Maximum stack size (default: 1)

**Returns:** `true` on success, `false` on validation failure

**Example:**
```gdscript
api.register_item({
    "id": "my_item",
    "name": "My Custom Item",
    "description": "Does something cool.",
    "category": "resource",
    "stack_size": 20
})
```

#### register_recipe(recipe_data: Dictionary) -> bool

Register a custom crafting recipe.

**Required fields:**
- `id` (string) - Recipe identifier
- `station` (string) - Crafting station: "workbench", "forge", "lab", etc.
- `inputs` (array) - Input items [{item_id, quantity}]
- `output` (object) - Output item {item_id, quantity}

**Optional fields:**
- `name` (string) - Display name
- `description` (string) - Recipe description
- `craft_time` (float) - Crafting duration in seconds (default: 5.0)

#### register_profession(profession_data: Dictionary) -> bool

Register a custom profession.

#### register_train_car(car_data: Dictionary) -> bool

Register a custom train car.

### Content Queries

#### get_item(id: String) -> Dictionary

Get item data by ID. Returns empty Dictionary if not found.

#### get_all_item_ids() -> Array[String]

Get all registered item IDs.

#### item_exists(id: String) -> bool

Check if an item exists.

---

## EventHooks Signals

Connect to EventHooks signals to respond to game events.

### Lifecycle Events

| Signal | Parameters | Description |
|--------|------------|-------------|
| `game_ready` | none | Game initialization complete |
| `game_paused` | none | Game paused |
| `game_resumed` | none | Game resumed |
| `game_exiting` | none | Game shutting down |

### Content Events

| Signal | Parameters | Description |
|--------|------------|-------------|
| `item_registered` | item_id, mod_id | Item registered |
| `recipe_registered` | recipe_id, mod_id | Recipe registered |
| `content_loaded` | content_type, count | Content batch loaded |

### Crafting Events

| Signal | Parameters | Description |
|--------|------------|-------------|
| `craft_started` | recipe_id, station_id | Crafting started |
| `craft_completed` | recipe_id, station_id, item_id | Crafting finished |
| `craft_failed` | recipe_id, reason | Crafting failed |

### Train Events

| Signal | Parameters | Description |
|--------|------------|-------------|
| `train_car_entered` | car_id, player_id | Player entered car |
| `train_car_exited` | car_id, player_id | Player exited car |
| `subsystem_state_changed` | car_id, subsystem, new_state | Subsystem changed |

### Expedition Events

| Signal | Parameters | Description |
|--------|------------|-------------|
| `expedition_started` | location_id | Expedition began |
| `expedition_ended` | location_id, success | Expedition finished |
| `escalation_changed` | old_level, new_level | Escalation changed |

### Example Usage

```gdscript
func _mod_init(api: ModAPI) -> void:
    EventHooks.game_ready.connect(_on_game_ready)
    EventHooks.craft_completed.connect(_on_craft_completed)

func _on_game_ready() -> void:
    print("Game is ready!")

func _on_craft_completed(recipe_id: String, station_id: String, item_id: String) -> void:
    print("Crafted: %s" % item_id)
```

### items.json

  "items": [
    {
      "id": "item_identifier",
      "name": "Display Name",
      "description": "Item description",
      "category": "resource",
      "stack_size": 50
    }
  ]
}
```

### recipes.json

```json
{
  "recipes": [
    {
      "id": "recipe_identifier",
      "name": "Recipe Name",
      "station": "workbench",
      "inputs": [
        {"item_id": "input_item", "quantity": 2}
      ],
      "output": {"item_id": "output_item", "quantity": 1},
      "craft_time": 5.0
    }
  ]
}
```

4. **Test loading** - Verify your mod loads without errors before distributing
5. **Document your mod** - Include a README.md explaining what your mod does

# Getting Started with Modding

Welcome to Off The Rails modding! This guide walks you through creating your first mod.

## Prerequisites

- Off The Rails installed
- Text editor (VS Code recommended)
- Basic understanding of JSON format

---

## Creating Your First Mod

### Step 1: Create the Mod Folder

Create a new folder for your mod in the game's mod directory:

**Windows:** `%APPDATA%/Godot/app_userdata/Off The Rails/mods/my_first_mod/`
**Linux:** `~/.local/share/godot/app_userdata/Off The Rails/mods/my_first_mod/`
**macOS:** `~/Library/Application Support/Godot/app_userdata/Off The Rails/mods/my_first_mod/`

### Step 2: Create mod.json

Every mod needs a `mod.json` manifest. Create this file:

```json
{
  "id": "my_first_mod",
  "version": "1.0.0",
  "name": "My First Mod",
  "description": "My first Off The Rails mod!",
  "author": "Your Name",
  "content_files": ["data/items.json"]
}
```

**Important:**
- `id` must be unique (lowercase, no spaces)
- `version` follows X.Y.Z format

### Step 3: Add Custom Items

Create the `data/` folder and add `items.json`:

```json
{

      "category": "tool",
      "stack_size": 1
    }
  ]
}
```

### Step 4: Test Your Mod

1. Launch Off The Rails
2. Check the console for `[ModLoader] Discovered 1 mods`
3. Your item should appear in the game!

| Error | Solution |

| "Mod not found" | Check folder name matches mod ID |
| "Invalid manifest" | Validate mod.json JSON syntax |
| "Content load failed" | Check YAML file syntax |
| "Script load failed" | Check GDScript syntax |
