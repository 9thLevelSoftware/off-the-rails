## Integration Pattern

```
PlayerController (CharacterBody2D)
       ↓ position
IsometricInteractionController (Node)
       ↓ queries
IsometricProximityDetector (RefCounted)
       ↓ finds nearest
InteractionStateMachine (RefCounted)
       ↓ signals
InteractionPromptDisplay (CanvasLayer child)
       ↓ shows
"Press E to interact"
```

## Y-Sort Considerations

Prompt must render above equipment but below player when player is behind.
**Solution**: Prompt as Control child of IsometricInteractionController (not CanvasLayer — that bypasses Y-sorting). Use explicit `z_index = 100` to render above equipment sprites.

## Pre-Execution Fixes Required

**PlayerController group membership**: PlayerController.gd must call `add_to_group("player")` in _ready(). Verify this exists or add it as Task 0.

## Task 3: IsometricInteractionController (adapter)

**File**: `src/interaction/adapters/isometric_interaction_controller.gd`

**Action**:
Create the main interaction controller that orchestrates the system.

Requirements:
- `class_name IsometricInteractionController` extends Node
- `@export var prompt_scene: PackedScene` — interaction_prompt.tscn
- `signal interaction_requested(interactable_id: String)` — Emitted when player presses interact
- `signal interactable_entered(interactable_id: String)`
- `signal interactable_exited()`
- Internal references:
  - `var _player: CharacterBody2D = null`
  - `var _detector: IsometricProximityDetector`
  - `var _state_machine: InteractionStateMachine`
  - `var _prompt: InteractionPromptDisplay = null`
  - `var _interactables: Dictionary = {}` — id → {node: Node2D, config: InteractableConfig}
- `func _ready()`:
  - Create detector and state machine instances
  - Instantiate prompt scene as child
  - Call deferred player lookup
- `func _physics_process(delta)`:
  - **Call `_state_machine.tick(delta)`** — Required for cooldown management
  - Update state machine with player position
  - Update prompt position if visible
- `func _unhandled_input(event)`:
  - Handle "interact" action press → call state machine try_interact()
- `func register_interactable(id: String, node: Node2D, config: InteractableConfig) -> void`
- `func unregister_interactable(id: String) -> void`
- Connect state machine signals to show/hide prompt

**Verify**: `grep -q "class_name IsometricInteractionController" src/interaction/adapters/isometric_interaction_controller.gd && grep -q "signal interaction_requested" src/interaction/adapters/isometric_interaction_controller.gd`

**Done**: IsometricInteractionController orchestrates interaction detection, input, and visual feedback.

## Task 4: EquipmentInteractable (workshop adapter)

**File**: `src/train/cars/workshop/adapters/equipment_interactable.gd`

**Action**:
Create adapter that makes EquipmentEntity interactable.

Requirements:
- `class_name EquipmentInteractable` extends Node2D
- `signal interaction_triggered(equipment_id: String, equipment_type: String)`
- `@export var equipment_id: String = ""`
- `@export var config: InteractableConfig`
- `var _entity: EquipmentEntity = null` — Set by WorkshopSpatialAdapter
- `func setup(entity: EquipmentEntity) -> void`:
  - Store entity reference
  - Set equipment_id from entity
  - Create default config if not exported (use `InteractableConfig.create_default()`)
  - Set position from entity.get_world_position()
  - **Use `call_deferred("_register_with_controller")`** for timing safety
- `func _register_with_controller() -> void`:
  - Find IsometricInteractionController in tree
  - Call `controller.register_interactable(equipment_id, self, config)`
- `func get_interaction_position() -> Vector2` — Returns position for prompt
- `func on_interact() -> void` — Called when player interacts
  - Emit interaction_triggered signal
  - Print placeholder feedback: "Interacted with {equipment_id}"
- Add self to "interactable" group on ready

**Verify**: `grep -q "class_name EquipmentInteractable" src/train/cars/workshop/adapters/equipment_interactable.gd && grep -q "call_deferred" src/train/cars/workshop/adapters/equipment_interactable.gd`

**Done**: EquipmentInteractable bridges EquipmentEntity to the interaction system with safe deferred registration.
