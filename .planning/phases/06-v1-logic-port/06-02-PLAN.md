  <name>Task 1: Add ContentRegistry accessor to GameState</name>
  <files>src/autoloads/game_state.gd</files>
  <action>
Add a ContentRegistry accessor to GameState so all systems can query game content through a single autoload:

1. Add a private variable to hold the registry reference:
   ```gdscript
   # --- Content Registry ---
   var _content_registry: ContentRegistry = null
   ```

2. Add an initialization method (call from game init or lazily):
   ```gdscript
   ## Initialize the content registry. Call once at game startup.
   func init_content_registry() -> bool:
       if _content_registry != null:
           return true
       _content_registry = ContentRegistry.new()
       return _content_registry.load_base_content()
   ```

3. Add a getter for other systems to access:
   ```gdscript
   ## Get the content registry. Initializes lazily if not yet loaded.
   func get_content_registry() -> ContentRegistry:
       if _content_registry == null:
           init_content_registry()
       return _content_registry
   ```

4. Add convenience accessors for common lookups:
   ```gdscript
   ## Get an item definition by ID.
   func get_item_data(item_id: String) -> ResourceItemData:
       return get_content_registry().get_item(item_id)
   
   ## Get a recipe definition by ID.
   func get_recipe_data(recipe_id: String) -> RecipeData:
       return get_content_registry().get_recipe(recipe_id)
   
   ## Check if an item ID is valid (exists in registry).
   func is_valid_item(item_id: String) -> bool:
       return get_content_registry().items.has_item(item_id)
   ```

5. Optionally add item validation to `add_to_inventory()`:
   ```gdscript
   # In add_to_inventory(), optionally warn if item not in registry:
   if _content_registry and not _content_registry.items.has_item(item_id):
       push_warning("GameState: Adding unknown item '%s' to inventory" % item_id)
   ```

<task type="auto">
  <name>Task 2: Wire RecipeRepository to prefer ContentRegistry with .tres fallback</name>
  <files>src/crafting/infrastructure/recipe_repository.gd</files>
  <action>
Update RecipeRepository to prefer ContentRegistry (JSON) but fall back to .tres files for backward compatibility.

**CRITICAL CONTEXT**: Plan-critique identified a data format mismatch:
- RecipeRepository currently loads .tres files from `res://src/data/recipes/`
- ContentRegistry (Phase 5) loads JSON from `res://data/base_recipes.json`
- These are PARALLEL systems that need reconciliation

**Integration Strategy**: Prefer ContentRegistry when available, fall back to .tres for:
- Development testing with individual .tres files
- Migration period until all recipes are in JSON

1. Add ContentRegistry preference at the top of the class:
   ```gdscript
   ## If true, prefer ContentRegistry over .tres files
   var _use_content_registry: bool = true
   ```

2. Modify `get_recipe()` to check ContentRegistry first:
   ```gdscript
   func get_recipe(recipe_id: String) -> RecipeData:
       # Prefer ContentRegistry if available
       if _use_content_registry and GameState and GameState.get_content_registry():
           var registry_recipe := GameState.get_content_registry().get_recipe(recipe_id)
           if registry_recipe:
               return registry_recipe
       
       # Fall back to local .tres cache
       _ensure_loaded()
       return _recipes.get(recipe_id, null)
   ```

3. Modify `get_all_recipes()` to merge sources:
   ```gdscript
   func get_all_recipes() -> Array[RecipeData]:
       var result: Array[RecipeData] = []
       var seen_ids: Dictionary = {}
       
       # ContentRegistry recipes first (higher priority)
       if _use_content_registry and GameState and GameState.get_content_registry():
           var registry := GameState.get_content_registry()
           for recipe in registry.recipes.get_all_recipes():
               result.append(recipe)
               seen_ids[recipe.id] = true
       
       # Add .tres recipes not already in ContentRegistry
       _ensure_loaded()
       for recipe_id in _recipes:
           if recipe_id not in seen_ids:
               result.append(_recipes[recipe_id])
       
       return result
   ```

4. Keep existing .tres loading as fallback — do NOT remove `load_all_recipes()`.

5. Add helper to check data source:
   ```gdscript
   ## Set whether to prefer ContentRegistry over .tres files.
   func set_use_content_registry(enabled: bool) -> void:
       _use_content_registry = enabled
   
   ## Check if ContentRegistry is being used.
   func is_using_content_registry() -> bool:
       return _use_content_registry and GameState and GameState.get_content_registry() != null
   ```
