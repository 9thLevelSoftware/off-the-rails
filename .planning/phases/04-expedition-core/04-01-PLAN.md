<objective>
Create the foundational escalation system that tracks threat level during expeditions. This system provides the core state management and signal contracts that Plans 04-02, 04-03, and 04-04 will build upon.
</objective>

<context>
## Existing Code
- `src/train/subsystems/subsystem.gd` — State machine pattern to follow
- `src/expedition/expedition.gd` — Scene script to integrate with
- `src/autoloads/game_state.gd` — Reference for signal patterns

## Design Reference
- `docs/design/systems/expeditions.md` — Escalation meter section
- Thresholds: Normal (0-25%), Elevated (26-50%), High (51-75%), Critical (76-99%), Overrun (100%)

## Architecture Constraints
- EscalationManager should be a Node added to expedition scene (not autoload)
- Follows Subsystem pattern: state property with setter, signals on change
- Must be testable in isolation (no hard dependencies on other systems)
</context>
