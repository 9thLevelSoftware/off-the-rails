## Task 1: RecipeData Resource Class

**Action**: Create `src/crafting/domain/recipe_data.gd` as a Resource class that holds recipe definition data.

```gdscript
class_name RecipeData
extends Resource

@export var id: String
@export var display_name: String
@export var category: String  # consumable, ammo, medical, etc.
@export var station: String   # workshop, armory, etc.
@export var inputs: Dictionary  # {resource_id: quantity}
@export var outputs: Dictionary # {item_id: quantity}
@export var craft_time: float   # seconds
@export var unlock_type: String # default, schematic_common, etc.
@export var profession_bonus: String  # profession that gets -25% time
@export var description: String
```

Include helper methods:
- `get_input_cost(resource_id: String) -> int`
- `get_output_quantity(item_id: String) -> int`
- `has_profession_bonus(profession_id: String) -> bool`

**Verify**: File exists at path, class_name is `RecipeData`, all fields present.

**Done**: RecipeData resource class created with all recipe fields and helper methods.

## Task 2: CraftJob Value Object

**Action**: Create `src/crafting/domain/craft_job.gd` as an immutable value object representing a queued crafting job.

```gdscript
class_name CraftJob
extends RefCounted

var job_id: int
var recipe: RecipeData
var total_time: float      # Calculated from recipe + modifiers
var elapsed_time: float    # Progress
var state: JobState        # QUEUED, IN_PROGRESS, COMPLETED, CANCELLED

enum JobState { QUEUED, IN_PROGRESS, COMPLETED, CANCELLED }
```

Include methods:
- `static func create(recipe: RecipeData, profession_bonus: float = 1.0) -> CraftJob`
- `func get_remaining_time() -> float`
- `func get_progress_percent() -> float`
- `func advance_time(delta: float) -> void`
- `func is_complete() -> bool`
- `func cancel() -> void`

Job should be created via factory method, not constructor, to enforce immutable recipe reference.

**Verify**: CraftJob can be instantiated, progress tracking works correctly.

**Done**: CraftJob value object created with state tracking and progress calculation.

## Task 3: CraftQueue Entity

**Action**: Create `src/crafting/domain/craft_queue.gd` as the queue entity managing ordered jobs.

```gdscript
class_name CraftQueue
extends RefCounted

var jobs: Array[CraftJob] = []
var max_slots: int = 1  # T1 station default
var _next_job_id: int = 1

signal job_added(job: CraftJob)
signal job_removed(job: CraftJob)
signal queue_changed()
```

Include methods:
- `func can_add_job() -> bool` — Check slot availability
- `func add_job(job: CraftJob) -> bool` — Add to queue
- `func remove_job(job_id: int) -> CraftJob` — Remove by ID
- `func get_active_job() -> CraftJob` — First job (currently crafting)
- `func get_queued_jobs() -> Array[CraftJob]` — All jobs after active
- `func clear() -> void`
- `func get_job_count() -> int`

Queue should emit signals when modified for UI binding.

**Verify**: Queue respects slot limits, jobs ordered correctly.

**Done**: CraftQueue entity created with slot management and signals.

## Task 4: RecipeValidator Use Case

**Action**: Create `src/crafting/domain/recipe_validator.gd` as use case for validating craft requests.

```gdscript
class_name RecipeValidator
extends RefCounted

## Validates whether a recipe can be crafted given current resources and station.

func can_craft(recipe: RecipeData, inventory: Dictionary, station_id: String) -> ValidationResult
func get_missing_resources(recipe: RecipeData, inventory: Dictionary) -> Dictionary
func calculate_craft_time(recipe: RecipeData, profession_id: String, station_tier: int) -> float
```

Create `ValidationResult` as inner class or separate file:
```gdscript
class ValidationResult:
    var success: bool
    var reason: String  # Empty if success
    var missing_resources: Dictionary  # {resource_id: missing_quantity}
```

Time calculation formula:
- Base time from recipe
- -25% if profession matches `profession_bonus`
- Station tier bonus: T1=0%, T2=-30%, T3=-60%, T4=-100% (V1 uses T1 only)

**Verify**: Validator correctly identifies missing resources, calculates time with bonuses.

**Done**: RecipeValidator use case created with validation and time calculation.
