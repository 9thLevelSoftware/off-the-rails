<objective>
Create the game source directory structure and implement the GameState autoload following the architecture sketch. This establishes the foundational file organization and core state management for all subsequent phases.
</objective>

src/

**Verify**: All 6 directories exist under `src/`

**Done when**: `ls -la src/` shows all subdirectories

---

## Task 2: Create GameState Autoload

**Action**: Implement `src/autoloads/game_state.gd` as the core state manager

**Instructions**:
1. Create a GDScript that extends Node
2. Include foundational state properties:
   - `campaign_phase: int` — Current campaign phase (1-5)
   - `current_location: String` — Current expedition location ID or "train"
   - `session_active: bool` — Whether a session is in progress
3. Include lifecycle signals:
   - `signal session_started`
   - `signal session_ended`
   - `signal location_changed(new_location: String)`
4. Add a `_ready()` function that initializes default state
5. Add basic methods: `start_session()`, `end_session()`, `change_location(loc: String)`

**Verify**: Script parses without errors via `mcp__godot-lsp__get_diagnostics`

**Done when**: `game_state.gd` exists with all signals, properties, and methods

---

## Task 3: Register Autoload in project.godot

**Action**: Configure GameState as an autoload singleton

**Instructions**:
1. Use `mcp__godot-mcp__manage_autoloads` to register:
   - Name: `GameState`
   - Path: `res://src/autoloads/game_state.gd`
   - Enabled: true
2. Verify registration via `mcp__godot-mcp__read_project_settings`
3. Confirm no conflicts with existing GDAIMCPRuntime autoload

**Verify**: project.godot contains GameState autoload entry

**Done when**: `mcp__godot-mcp__read_project_settings` shows GameState in autoload section

---

## Task 4: Runtime Autoload Verification

**Action**: Verify GameState autoload initializes correctly at runtime

**Instructions**:
1. Create a minimal test scene `src/test_autoload.tscn`:
   - Root: Node named "Test"
   - Script with `_ready()` that prints `print("GameState.campaign_phase = ", GameState.campaign_phase)`
2. Run the test scene via `mcp__godot-mcp__run_project` or Godot editor
3. Check output for successful GameState access
4. Verify autoload initialization order:
   - GDAIMCPRuntime should initialize first (existing)
   - GameState second (no dependencies on GDAIMCPRuntime)
5. Clean up test scene after verification (or keep for regression testing)

**Verify**: Console output shows `GameState.campaign_phase = 1` (or default value)

**Done when**: GameState is confirmed accessible from _ready() of another scene
