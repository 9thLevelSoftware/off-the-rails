<objective>
Create the loot container system that gives players a reason to explore and stay in the expedition longer. Containers should feel satisfying to open and optionally contribute to escalation.

**CRITICAL**: Also extend InteractionController to detect expedition interactables, not just train cars.
</objective>

<context>
## Prior Plan Output
- EscalationManager from 04-01 (for optional loot escalation trigger)

## Existing Patterns
- `src/train/interaction/interactable.gd` — Interface to extend
- `src/train/interaction/interaction_controller.gd` — MUST BE EXTENDED (currently only searches "train_car" group)

## Critique Finding (BLOCKER)
InteractionController._find_nearest_train_car() searches only "train_car" group and returns TrainCar type. LootContainers will NOT be detected without extending this system.

**Solution**: Add "interactable" group search to InteractionController, or create unified interactable lookup that checks both train_car and a new "expedition_interactable" group.

## Design Reference
- `docs/design/systems/expeditions.md` — Loot containers mentioned
- Opening sealed areas: +5-10 escalation

## V1 Scope
- Simple item: name + quantity (no inventory system yet)
- Visual feedback: container "opens" (change mesh or visibility)
- Audio: deferred to later phase
- No procedural loot tables (hardcoded for V1)
</context>

<task id="7">
<action>
Cache escalation manager in _ready() with fallback:
```gdscript
func _ready() -> void:
    super._ready()  # Interactable._ready() if any
    add_to_group("interactable")  # Ensure detectable
    call_deferred("_cache_escalation_manager")

func _cache_escalation_manager() -> void:
    # Try to find EscalationManager in expedition scene
    var expedition = get_tree().get_first_node_in_group("expedition")
    if expedition:
        _escalation_manager = expedition.get_node_or_null("EscalationManager")
    if not _escalation_manager:
        push_warning("LootContainer: No EscalationManager found, sealed containers won't trigger escalation")
```
</action>
